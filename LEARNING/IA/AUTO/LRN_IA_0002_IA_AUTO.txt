WBS — SKILL IA/AUTOMATIZACIÓN v1.0

Nombre: AUTO.AI.OPERATOR
Propósito: pasar de usuario a operador que diseña, integra y entrega automatizaciones con impacto

00.00 PRINCIPIO RECTOR
IA sin integración = juguete
Automatización sin medición = irrelevante
Cada módulo termina con un flujo funcionando (aunque sea pequeño)
01.00 INPUT CONTRACT
01.01 Objetivo
diseñar y ejecutar automatizaciones reales (personal o negocio)
output: 3–5 flujos productivos + documentación + métricas
01.02 Restricciones
6–8 semanas
5–8 hrs/semana
entorno: Windows + PowerShell (tu caso), APIs públicas
01.03 Nivel
base técnica media (scripts/CLI asistidos por IA)
01.04 Nivel objetivo por módulo
M1–M2 → L2
M3–M4 → L3
M5 → L4
02.00 MAPA DEL CONOCIMIENTO
M1 Fundamentos IA aplicada (L2)
qué puede / no puede hacer
tipos de tareas (texto, clasificación, extracción)
M2 Prompt Engineering operativo (L3)
inputs estructurados
control de output
validación
M3 Automatización (scripts + APIs) (L3)
requests a APIs
manejo de archivos
flujos CLI
M4 Integración (L3)
conectar servicios (correo, archivos, APIs)
orquestación simple
M5 Casos reales (L4)
diseñar soluciones completas
optimizar
documentar
03.00 CICLO OPERATIVO (POR MÓDULO)
03.01 INPUT

03.02 PROCESAMIENTO

03.03 RECUPERACIÓN

03.05 FEEDBACK

03.06 ITERACIÓN
