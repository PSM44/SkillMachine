00.20_CONTEXTO
==========

Objetivo:
- La IA debe poder “hacer de todo” dentro del proyecto,
  pero debe quedar imposibilitada de salir de ese perímetro.

==========
01.00_DEFINICIONES
==========

01.10_PROJECT_ROOT
- Carpeta raíz canónica de un proyecto.
- Todas las operaciones del executor deben resolverse a paths bajo este root.

01.20_ALLOWLIST
- Lista de subpaths permitidos para escritura (y opcionalmente lectura).

01.30_DENYLIST
- Subpaths o patrones prohibidos incluso dentro del root (ej: secrets).

==========
02.00_CONTRATO_DURO_DE_BOUNDARY
==========

02.10_REGLA_CENTRAL
- Toda operación FS debe pasar por una función:
  ResolveAndVerifyPath(path)
- Si ResolveAndVerifyPath detecta que el path resuelto NO está bajo PROJECT_ROOT:
  - FAIL inmediato,
  - no se realiza ninguna acción,
  - se registra TROUBLE.

02.20_NORMALIZACION
- Resolver:
  - .. , symlinks (si aplica), junctions,
  - case normalization (Windows),
  - path canonical.
- Evitar bypasses por paths relativos o enlaces.

==========
03.00_ALLOWLIST
==========

03.10_DEFAULT
- Escritura permitida en:
  - \01_HUMAN\
  - \02_CODE\
  - \03_ARTIFACTS\
  - \AI_ORCH\ (si está dentro del proyecto)
- Lectura permitida en todo el root EXCEPTO denylist.

03.20_MODO_RESTRICTIVO_(OPCIONAL)
- Escritura solo en:
  - \02_CODE\
  - \03_ARTIFACTS\
- HUMAN/SKILLS solo se escriben con aprobación explícita (modo governance).

==========
04.00_DENYLIST
==========

04.10_SECRET_PATTERNS
Bloquear lectura/escritura:
- **\*.key
- **\*.pem
- **\*.pfx
- **\*.kdbx
- **\.env
- **\secrets\**
- **\credentials\**

04.20_ZONAS_SENSIBLES_DEL_PROYECTO
- \03_ARTIFACTS\SNAPSHOTS\  (escritura solo por el orquestador)
- \.git\objects\            (no tocar directo)

==========
05.00_POLITICA_DE_SCRIPTS_(RUNNER_ALLOWLIST)
==========

05.10_REGLA
- Executor no ejecuta comandos OS arbitrarios.
- Solo ejecuta scripts allowlisted desde:
  \AI_ORCH\02_TOOLS\ o \03_ARTIFACTS\RUNNERS\

05.20_ALLOWLIST_EJEMPLO
- RADAR.ps1 (si aplica)
- Test suite (ej: pytest, npm test) pero invocado por runner controlado.

05.30_PARAMETROS_CONTROLADOS
- Los parámetros de scripts son validados:
  - paths deben estar dentro del PROJECT_ROOT,
  - no se aceptan strings que permitan “escape” (ej: ; & |).

==========
06.00_POLITICA_DE_DELETES_(QUARANTINE)
==========

- Todo delete => move a QUARANTINE dentro del root (ver EXEC_POLICY).
- Mantener meta:
  - timestamp,
  - motivo,
  - task id,
  - checkpoint id.

==========
07.00_EVIDENCIA_Y_LOGGING
==========

Registrar por cada acción:
- actor (executor)
- operation (create/edit/move/quarantine)
- path_before / path_after
- hash_before / hash_after (si aplica)
- resultado OK/FAIL

==========
08.00_CASOS_DE_FALLO_(TROUBLE)
==========

- Boundary violation => TROUBLE severidad MEDIA mínimo.
- Attempt secret access => TROUBLE severidad ALTA (riesgo).
- Script runner intento no allowlisted => FAIL + TROUBLE.
