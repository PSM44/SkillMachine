00.20_CONTEXTO_Y_PRINCIPIOS
==========

00.20.01_CONTEXTO
- ChatGPT Plus NO cubre API.
- AI_ORCH usa cloud vía API, por lo tanto hay gasto variable.
- Si no se controla, el costo se dispara en ejecución frecuente.

00.20.02_PRINCIPIOS
- Control por DEFAULT: budgets + hard stop.
- “Preflight” siempre antes de ejecutar (si hay escritura en FS).
- Logging y auditoría obligatorios.
- Estimación MVP: suficiente para no volarse la tarjeta, no para precisión contable.

==========
01.00_DEFINICIONES
==========

01.10_BUDGET
- Límite de gasto permitido para un periodo (día/semana/mes).

01.20_PREFLIGHT
- Estimación de tokens y costo ANTES de llamar al modelo.

01.30_HARD_STOP
- Bloqueo automático: si budget excedido, se rechaza la tarea.

01.40_TASK_ID
- ID único por tarea (preferible: timestamp + secuencia + id proyecto).

==========
02.00_UNIDADES_DE_COSTO
==========

02.10_TOKENS
- input_tokens: tokens enviados al modelo.
- output_tokens: tokens generados por el modelo.

02.20_USD
- costo_usd = (input_tokens * rate_in) + (output_tokens * rate_out)

02.30_COSTO_POR_TAREA
- Suma de todas las llamadas a modelos para un TASK_ID.

02.40_COSTO_POR_PROYECTO
- Suma de todas las tareas asociadas a PROJ_ID en un periodo.

==========
03.00_BUDGETS
==========

03.10_BUDGET_DIARIO
- default recomendado: 5–15 USD/día (MVP)
- regla: si budget diario agotado => hard stop, salvo override humano.

03.20_BUDGET_SEMANAL
- default recomendado: 30–80 USD/semana (MVP)
- objetivo: suavizar picos diarios.

03.30_BUDGET_MENSUAL
- default recomendado: 120–300 USD/mes (uso fuerte)
- nota: depende de cuánta automatización estés corriendo.

03.40_OVERRIDES
- Solo HUMANO puede aumentar budget.
- Override debe quedar loggeado.

==========
04.00_PREFLIGHT
==========

04.10_CUANDO_APLICA
- Siempre antes de:
  - “task/apply” que modifique FS
  - “task/plan” que pida contexto grande (RADAR_CORE o FULL)

04.20_METODO_MVP_DE_ESTIMACION
- Estimación rápida:
  - input_tokens_est = tamaño_contexto_chars / 4 (aprox)
  - output_tokens_est = default según tipo de tarea (ver 05.ROUTING_POLICY)
- Riesgo: aproximación. Aceptable para budgets.

04.30_PREFLIGHT_OUTPUT
Debe retornar:
- tokens_est_in
- tokens_est_out
- costo_est_usd (por modelo)
- riesgo (ALTO/MEDIO/BAJO)
- recomendación: usar modelo económico / reducir contexto / chunk.

==========
05.00_HARD_STOP
==========

05.10_REGLA
- Si costo_actual_periodo + costo_est_tarea > budget_periodo:
  - rechazar ejecución (FAIL controlado)
  - sugerir alternativa (reducir contexto / modelo económico / esperar próximo periodo)

05.20_FALLBACK_SI_RECHAZA
- “plan-only” permitido sin ejecución
- “dry-run patch” permitido sin aplicar FS

==========
06.00_ALERTAS_Y_UMBRALES
==========

06.10_UMBRALES_RECOMENDADOS
- 50% budget: aviso
- 80% budget: aviso fuerte
- 100% budget: hard stop

06.20_CANAL
- Output local en logs
- Notificación en IDE (si existe integración)
- Resumen diario opcional

==========
08.00_MODELO_DE_ESTIMACION_(MVP)
==========

08.10_BASELINES_POR_TIPO_DE_TAREA
- PLAN: output_tokens_est 400–1200
- CODEGEN: 800–2500
- REFACTOR: 1500–5000
- DOC_CANON: 1200–4000
- RADAR_SUMMARY: 600–2000

08.20_AJUSTES
- Si contexto = RADAR_CORE: multiplicar input_est x2
- Si contexto = RADAR_FULL: multiplicar input_est x4 (desaconsejado para MVP)
