TEMPLATE — GRC MINIMUM VIABLE PACKAGE (GRC-MVP)
==========================================

ID_UNICO..........: TEMPLATE.GRC.MVP.0001
VERSION...........: v0.1
ESTADO............: ACTIVE
FECHA_CREACION....: YYYY-MM-DD
FECHA_MODIFICACION: YYYY-MM-DD

PROPOSITO.........:
Estándar mínimo obligatorio para que un artefacto sea considerado GRC (y no solo una "skill con etiqueta").
Este template define los campos mandatorios para: POLICY, CONTROL, EVIDENCE MODEL, VALIDATION, SEVERITY y CHANGE CONTROL.

JERARQUIA_CANONICA:
1) HUMAN (manda; propósito/espíritu/horizonte)
2) GRC (manda operativamente; reglas obligatorias + enforcement)
3) SKILLS (capacidades; cómo ejecutar)
4) USECASE (paquetes aplicados; deployment/runbook)
5) TOOLS/ARTEFACTOS/LOGS (operación)

REGLA_DURA:
- Si falta algún bloque obligatorio de este template, el artefacto NO califica como GRC.
- Un GRC siempre debe ser ENFORCEABLE: verificable por control y con evidencia mínima.

SCOPE:
Este template aplica a:
- policies
- controls
- risks
- evidence models
- mappings
- registries GRC
cuando se declaran como GRC mandatorio.

FUERA_DE_SCOPE:
- Skills tácticas sin enforcement
- Documentos informativos sin validación
- Outputs transitorios (logs, temp, raw)

10. ACCEPTANCE CRITERIA (MANDATORY)
------------------------------------------

ACCEPTANCE_CRITERIA:
- [ ] Metadata completa
- [ ] Trace a HUMAN definido
- [ ] Policy (si aplica) escrita sin ambigüedad
- [ ] Control definido con PASS/FAIL claro
- [ ] Evidence model definido (mínimos + formato + ubicación)
- [ ] Validation definida con output verificable
- [ ] Severity justificada
- [ ] Change control definido
- [ ] Versionado definido
- [ ] Ready para enforcement (no solo texto)

DEFINITION_OF_DONE:
DONE solo si:
- Archivo existe
- Validación PASS (si aplica)
- Evidencia registrada
- Version incrementada si cambió lo canónico
