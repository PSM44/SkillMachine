01.00_OBJETIVO
==========

Establecer un estándar mínimo obligatorio para que todo proyecto:

- tenga contexto para IA
- tenga continuidad
- tenga trazabilidad
- sea recuperable por terceros

==========
02.00_ARTEFACTOS_OBLIGATORIOS
==========

Todo proyecto DEBE crear:

1. BATON
2. RADAR
3. HUMAN
4. WhoAmI

Sin estos 4 elementos:
→ el proyecto se considera NO-OPERABLE

==========
03.00_ORDEN_DE_CREACION
==========

Orden obligatorio:

01 → HUMAN
02 → WhoAmI
03 → BATON
04 → RADAR

Motivo:

- HUMAN define contexto base
- WhoAmI define estado actual
- BATON sintetiza para IA
- RADAR mapea estructura

==========
04.00_REGLAS_OPERATIVAS
==========

04.10_HUMAN

- Documento explicativo completo
- No resumido
- Didáctico
- Puede tener redundancia

04.20_WHOAMI

- Estado actual del proyecto
- Qué es
- Qué hace
- En qué estado está
- Sin jerga técnica innecesaria

04.30_BATON

- Snapshot operativo
- Orientado a IA
- Derivado de HUMAN + WhoAmI

04.40_RADAR

- Index obligatorio
- Core opcional
- Full solo bajo demanda

==========
05.00_VALIDACION
==========

Antes de considerar proyecto válido:

- Existe HUMAN
- Existe WhoAmI
- Existe BATON
- Existe RADAR_INDEX

Si falta alguno:
→ FAIL

==========
06.00_ANTIPATRONES
==========

- Crear proyecto sin BATON
- Crear proyecto sin WhoAmI
- Usar solo RADAR sin HUMAN
- Generar BATON sin HUMAN previo
- RADAR como sustituto de contexto
