02.20_MODELO_ECONOMICO_(COSTO)

02.30_REGLA
- No más de 2 modelos en MVP para estabilidad:
  - principal
  - económico

==========
03.00_MATRIZ_ROUTING
==========

03.10_PLAN_ARCH
- modelo: principal
- contexto: L0 -> L1 (si necesita)
- salida: plan + riesgos + estimate

03.20_CODEGEN
- modelo: principal si afecta núcleo; económico si es boilerplate
- contexto: L1
- salida: patch + archivos nuevos

03.30_REFACTOR
- modelo: principal
- contexto: L1 -> L2 (si el repo lo exige)
- salida: patch + pasos de validación

03.40_DEBUG_TROUBLE
- modelo: principal (mejor razonamiento)
- contexto: L1 (logs + snippet)
- salida: hipótesis única + cambio mínimo + prueba (anti-shotgun)

03.50_DOC_CANON
- modelo: económico para docs mecánicas; principal si es doctrina/arquitectura
- contexto: L0/L1
- salida: secciones completas con FILE_CONTENT

03.60_RADAR_NAV
- modelo: económico
- contexto: L0 (RADAR_INDEX)
- salida: “dónde está X” + paths

03.70_COST_REPORT
- modelo: económico
- contexto: logs sqlite/txt
- salida: reporte.

==========
04.00_ESCALAMIENTO_DE_CONTEXTO
==========

04.10_REGLA_DE_ESCALAMIENTO
- Iniciar siempre en L0 o L1.
- Subir a L2 solo si:
  - el modelo lo pide explícitamente,
  - o falló el intento anterior por falta de contexto.
- L3 solo por auditoría, nunca por defecto.

04.20_REDUCCION_DE_CONTEXTO
- Si costo/tiempo sube:
  - reducir a L1 (snippets),
  - o usar “chunking” del CORE.

==========
05.00_FALLBACK_(MVP_MANUAL)
==========

- Si el modelo principal falla:
  - se registra FAIL + TROUBLE
  - el humano decide reintentar con:
    - el económico
    - o el principal con más contexto
- No fallback automático en MVP.

==========
06.00_APPLY_VS_PLAN_ONLY
==========

06.10_PLAN_ONLY_DEFAULT
- Toda tarea parte en plan-only.
- Apply solo si el plan fue aprobado por humano.

06.20_APPLY_AUTOMATICO_(OPCIONAL_FUTURO)
- Se habilita solo cuando:
  - budgets y sandbox están maduros,
  - tests mínimos existen,
  - incident rate es bajo.

==========
07.00_ANTIPATRONES
==========

- FULL context para todo.
- Usar modelo caro para navegación trivial.
- Apply sin plan aprobado.
- Más de 2 modelos en MVP.
