00.20_OBJETIVO_Y_REGLAS_BASE
==========

Objetivo:
Permitir que una IA seleccione el set mínimo correcto de skills
para resolver un requerimiento usando este menú como catálogo de referencia.

Reglas base:
- La IA solo puede recomendar skills existentes en este menú.
- La IA no debe inventar skills inexistentes.
- La IA debe priorizar el set mínimo suficiente.
- La IA debe detectar skills innecesarios o redundantes.
- Si no existe skill suficiente, debe declararlo explícitamente.

==========
01.00_REGLAS_DE_USO_DEL_MENU
==========

- Este archivo se adjunta junto al prompt del usuario.
- El usuario puede adjuntar además 0..N skills concretos.
- La IA debe evaluar cobertura del set adjunto contra el requerimiento.
- Si faltan skills, debe pedirlos explícitamente desde este menú.
- Si el menú no cubre el gap, debe indicarlo como candidato a mejora del sistema.

05.00_REGLAS_PARA_RECOMENDAR_SKILLS_ADICIONALES
==========

La IA debe recomendar skills adicionales solo si:
- mejoran cobertura de forma clara
- reducen riesgo de error
- evitan interpretación ambigua
- el skill adjunto por el usuario es insuficiente o incorrecto

La IA no debe recomendar skills por mera afinidad temática.

==========
06.00_REGLAS_PARA_DETECTAR_REDUNDANCIA
==========

La IA debe marcar skills como innecesarios si:
- no aportan al caso
- duplican función de otro skill ya adjunto
- agregan ruido sin aumentar precisión operativa

==========
07.00_REGLAS_PARA_DETECTAR_GAPS
==========

Si el requerimiento exige una capacidad que no está cubierta por skills existentes:
- la IA debe decirlo explícitamente
- no debe inventar skill nuevo
- debe proponer que el caso sea revisado posteriormente
