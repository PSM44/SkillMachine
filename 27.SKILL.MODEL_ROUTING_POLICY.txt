00.20_OBJETIVO
==========

Estandarizar el uso de modelos para:
- evitar improvisación
- reducir costo
- maximizar ejecución local
- asegurar calidad mediante GRC

==========
01.00_PRINCIPIOS_DE_ROUTING
==========

01.10_SEPARACION_DE_ROLES
- PLAN: modelo fuerte (razonamiento)
- ACT: modelo rápido (ejecución)
- VALIDATE: modelo confiable (verificación)

01.20_LOCAL_FIRST
Priorizar modelos locales (Ollama) para ejecución.

01.30_GRC_ALWAYS_ON
GRC obligatorio en todas las fases.

01.40_COSTO_CONTROLADO
Cloud solo cuando local no cumple.

==========
03.00_MODO_OPERATIVO
==========

03.10_PLAN

Objetivo:
- diseñar solución
- detectar riesgos
- definir cambios

Modelo:
- qwen3-coder:30b (local)
- Claude (si falla local)

Output:
- corto, estructurado
- sin código completo

03.20_ACT

Objetivo:
- ejecutar cambios
- generar código
- modificar archivos

Modelo:
- qwen2.5-coder:7b
- deepseek-coder:6.7b

Output:
- diff exacto
- comandos ejecutables

03.30_VALIDATE

Objetivo:
- verificar consistencia
- detectar errores
- confirmar cumplimiento

Modelo:
- Claude Code

==========
06.00_REGLAS_DE_COSTO
==========

- No usar Claude para tareas simples
- No usar 30B para cambios triviales
- No usar cloud sin justificación

==========
07.00_ESCALAMIENTO
==========

Escalar a cloud cuando:

- modelo local falla 2 veces
- output inconsistente
- tarea crítica

==========
08.00_ANTIPATRONES
==========

- usar siempre el modelo más grande
- ejecutar sin PLAN
- no validar
- mezclar modelos sin control
